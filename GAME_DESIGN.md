# Bullet Heaven - 게임 기획서

## 1. 게임 개요

- **장르**: 뱀파이어 서바이벌 류 (Bullet Heaven / Auto-shooter)
- **플랫폼**: 웹 브라우저 (HTML5 Canvas + JavaScript)
- **조작**: WASD/방향키로 이동, 공격은 자동
- **목표**: 스테이지 1~10을 클리어하며 무기를 수집하고 강화하여 살아남기

---

## 2. 스테이지 구성 (1~10)

| 스테이지 | 시간 | 테마 | 보스 | 특징 |
|---------|------|------|------|------|
| 1 | 60초 | 숲 | 없음 | 튜토리얼, 기본 적만 등장 |
| 2 | 75초 | 숲 | 없음 | 빠른 적 추가 |
| 3 | 90초 | 묘지 | 미니보스(거대 스켈레톤) | 첫 미니보스 |
| 4 | 90초 | 묘지 | 없음 | 원거리 적 추가 |
| 5 | 105초 | 성 외곽 | 보스(네크로맨서) | 중간보스, 적 소환 능력 |
| 6 | 105초 | 성 내부 | 없음 | 고체력 적(갑옷 기사) 추가 |
| 7 | 120초 | 성 내부 | 미니보스(다크나이트) | 돌진 패턴 |
| 8 | 120초 | 지하 던전 | 없음 | 폭발 적 추가, 밀집 스폰 |
| 9 | 135초 | 지하 던전 | 미니보스(리치) | 범위 공격, 디버프 |
| 10 | 150초 | 왕좌의 방 | 최종보스(뱀파이어 로드) | 다단계 보스전 |

### 난이도 곡선

- **적 스폰 속도**: 스테이지당 15% 증가
- **적 체력**: 스테이지당 20% 증가
- **적 이동속도**: 스테이지당 5% 증가
- **동시 출현 적 수 상한**: 스테이지 1(30) → 스테이지 10(150)

## 3. 플레이어

| 스탯 | 기본값 | 설명 |
|------|--------|------|
| HP | 100 | 피격 시 감소, 0이면 게임오버 |
| 이동속도 | 3 | 기본 이동 |
| 경험치 | 0 | 적 처치 시 획득, 레벨업에 사용 |

### 레벨업 시스템

- 적 처치 → 경험치 보석 드롭 → 수집 → 레벨업
- 레벨업 시 **3가지 선택지** 중 1개 선택 (스탯 강화)
  - HP 회복 +20
  - 이동속도 +10%
  - 공격력 +15%
  - 공격속도 +10%
  - 투사체 크기 +15%
  - 투사체 수 +1

---

## 4. 무기 시스템

### 무기 획득 방식

- 스테이지 진행 중 **무기 아이템**이 맵에 랜덤 스폰 (30~45초 간격)
- 아이템에 접촉하면 획득
- 이미 보유 중인 무기와 같은 종류면 **업그레이드** (최대 레벨 5)
- 새 무기면 추가 장착 (최대 **4개** 동시 장착)
- 슬롯이 꽉 차면 교체 여부 선택

### 무기 목록

| 무기 | 기본 공격 | Lv5 효과 | 발사방식 |
|------|----------|----------|---------|
| **단검** (시작 무기) | 가장 가까운 적에게 투사체 1개 | 투사체 3개, 관통 | 자동 조준 |
| **성수** | 주변 원형 범위 데미지 | 범위 2배, 지속 데미지 영역 | 자기 주변 |
| **도끼** | 위로 던져 포물선 낙하 | 3개 동시, 크기 증가 | 랜덤 방향 |
| **마법봉** | 랜덤 적에게 호밍 탄 | 동시 3발, 폭발 효과 | 호밍 |
| **십자가** | 부메랑처럼 왕복 | 2개 교차 회전, 크기 증가 | 회전 |
| **번개** | 랜덤 적에게 즉시 번개 | 체인 라이트닝 3회 | 즉발 |
| **마늘** | 근접 밀어내기 + 데미지 | 넉백 강화, 범위 2배 | 자기 주변 |
| **화염병** | 지면에 불 영역 생성 | 영역 크기 2배, 지속시간 증가 | 랜덤 위치 |

### 업그레이드 단계별 강화

- Lv1: 기본
- Lv2: 데미지 +30%
- Lv3: 공격속도 +25%
- Lv4: 투사체/범위 증가
- Lv5: 특수 효과 해금

## 5. 적 종류

| 적 | 등장 스테이지 | 행동 | 체력 | 데미지 |
|----|-------------|------|------|--------|
| 좀비 | 1~ | 느리게 직선 추적 | 낮음 | 5 |
| 박쥐 | 2~ | 빠르게 추적 | 매우 낮음 | 3 |
| 스켈레톤 | 3~ | 보통 추적 | 보통 | 8 |
| 유령 | 4~ | 원거리 탄 발사 | 낮음 | 10(탄) |
| 갑옷 기사 | 6~ | 느리지만 고체력 | 높음 | 15 |
| 폭발 슬라임 | 8~ | 근접 시 자폭 | 보통 | 25(폭발) |

### 보스

- **거대 스켈레톤 (스테이지 3)**: 넓은 범위 휘두르기, 느림
- **네크로맨서 (스테이지 5)**: 주기적으로 좀비 소환, 원거리 공격
- **다크나이트 (스테이지 7)**: 돌진 공격, 고체력
- **리치 (스테이지 9)**: 범위 마법, 플레이어 감속 디버프
- **뱀파이어 로드 (스테이지 10)**: 3단계 변신, 흡혈 공격, 박쥐 소환

## 7. UI/UX

- **HUD**: HP바, 경험치바, 현재 레벨, 장착 무기 슬롯, 스테이지/타이머
- **스테이지 클리어**: 타이머 종료 시 자동 다음 스테이지 진입 (스탯/무기 유지)
- **게임오버**: HP 0 → 결과 화면 (도달 스테이지, 처치 수, 플레이 시간)
- **시작 화면**: 게임 시작 버튼

## 8. 기술 스택

## 9. 아키텍처 설계 원칙

### 객체지향 + 인터페이스 기반 설계

모든 게임 요소를 **독립적인 클래스(객체)**로 분리하고, **공통 인터페이스(베이스 클래스)**를 통해 확장 가능하게 설계한다. 새로운 적, 무기, 아이템, 스테이지를 추가할 때 기존 코드를 수정하지 않고 새 클래스만 추가하면 되도록 한다.

### 베이스 클래스 인터페이스 규약

#### Entity (모든 게임 오브젝트의 부모)
```
필수 구현:
  - update(dt)        : 매 프레임 로직 업데이트
  - render(ctx)       : 캔버스에 그리기
  - getBounds()       : 충돌 판정 영역 반환
  - onDestroy()       : 제거 시 정리 작업
```

#### Enemy (적 베이스)
```
Entity를 상속하며 추가 구현:
  - hp, speed, damage  : 기본 스탯 (생성자에서 정의)
  - movementPattern()  : 이동 패턴 (추적, 순찰, 돌진 등)
  - onDeath()          : 사망 시 드롭/이펙트 처리
  - getDropTable()     : 드롭 테이블 반환
```

#### Boss (보스 베이스)
```
Enemy를 상속하며 추가 구현:
  - phases[]           : 페이즈별 행동 패턴
  - currentPhase       : 현재 페이즈
  - specialAttack()    : 특수 공격
  - onPhaseChange()    : 페이즈 전환 시 행동
```

#### Weapon (무기 베이스)
```
필수 구현:
  - name, level, maxLevel   : 무기 정보
  - cooldown, damage        : 기본 스탯
  - attack(player, enemies) : 공격 실행 (투사체 생성 등)
  - upgrade()               : 레벨업 시 강화 적용
  - getLevelDescription()   : 현재 레벨 설명 텍스트
```

#### Item (아이템 베이스)
```
Entity를 상속하며 추가 구현:
  - onPickup(player)   : 플레이어가 접촉 시 효과 적용
  - duration           : 존재 시간 (일정 시간 후 소멸)
```

#### Stage (스테이지 베이스)
```
필수 구현:
  - stageNumber, duration   : 스테이지 번호, 제한시간
  - spawnConfig             : 적 스폰 설정 (종류, 비율, 간격)
  - bossConfig              : 보스 등장 설정
  - getBackground()         : 배경 테마 반환
  - onStart() / onClear()   : 스테이지 시작/클리어 시 처리
```

### 레지스트리 패턴 (확장 용이성)

새로운 요소 추가 시 레지스트리에 등록만 하면 게임에 자동 반영:

```javascript
// 새 적 추가 예시
// 1. EvilMage.js 파일 생성 (extends Enemy)
// 2. EnemyRegistry에 등록
EnemyRegistry.register('evilMage', EvilMage);

// 새 무기 추가 예시
// 1. Scythe.js 파일 생성 (extends Weapon)
// 2. WeaponRegistry에 등록
WeaponRegistry.register('scythe', Scythe);
```

### 설계 원칙 요약

1. **단일 책임**: 각 클래스는 하나의 역할만 담당
2. **개방-폐쇄**: 기존 코드 수정 없이 새 클래스 추가로 확장
3. **인터페이스 분리**: 베이스 클래스가 명확한 계약(메서드)을 정의
4. **레지스트리 패턴**: 팩토리를 통한 동적 객체 생성으로 하드코딩 최소화
5. **데이터 주도**: 스테이지/적/무기 밸런스 값은 데이터 파일로 분리하여 수치 조정 용이
