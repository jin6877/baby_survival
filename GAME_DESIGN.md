# Baby Survival - 게임 기획서

## 1. 게임 개요

- **장르**: 뱀파이어 서바이벌 류 (Auto-shooter / Bullet Heaven)
- **플랫폼**: 웹 브라우저 (HTML5 Canvas + JavaScript)
- **화면 방향**: 가로형 (Landscape) 모바일 최적화
- **조작**: 가상 조이스틱(좌측)으로 이동, 공격은 자동
- **목표**: 세균, 바이러스, 위험 요소들을 피하며 엄마/아빠의 도움으로 살아남고 아기를 성장시키기

---

## 2. 핵심 컨셉

> **"아기가 살아남는다!"**
> 아기는 적들의 공격을 피하며 젖병으로 체력을 회복하고, 엄마와 아빠를 무기로 사용하여 세균과 바이러스를 처치한다.
> 스테이지를 클리어할수록 아기가 성장하여 새로운 모습으로 변한다.

---

## 3. 아기 성장 단계 (플레이어 성장)

아기는 스테이지 클리어 누적에 따라 외형과 능력이 변한다.

| 단계 | 해금 조건 | 외형 | 이동 방식 | 특징 |
|------|----------|------|----------|------|
| **1단계: 안겨있는 아기** | 시작 | 포대기에 쌓인 아기 | 둥둥 떠다님 | 가장 느리지만 무적 판정 살짝 큼 |
| **2단계: 기는 아기** | 스테이지 2 클리어 | 네발로 기는 아기 | 기어서 이동 | 이동속도 소폭 증가 |
| **3단계: 일어서는 아기** | 스테이지 4 클리어 | 비틀거리며 서있는 아기 | 뚜벅뚜벅 | 이동속도 증가, 점프 회피 해금 |
| **4단계: 유치원생** | 스테이지 6 클리어 | 유치원 가방 멘 아이 | 뛰어다님 | 이동속도 크게 증가, 특수 스킬 해금 |
| **5단계: 초등학생** | 스테이지 8 클리어 | 책가방 멘 초딩 | 빠르게 달림 | 이동속도 최대, 더블 무기 슬롯 해금 |
| **6단계: (보너스) 청소년** | 스테이지 10 클리어 | 교복 입은 중학생 | 전력질주 | 엔딩 전용 외형 |

### 성장 연출
- 단계 변화 시 **성장 컷씬** 재생 (2~3초 간단한 애니메이션)
- 각 단계마다 기본 스탯 보너스 적용

## 4. 스테이지 구성 (1~10)

| 스테이지 | 시간 | 배경 | 보스 | 테마 |
|---------|------|------|------|------|
| 1 | 60초 | 아기 방 | 없음 | 튜토리얼, 세균만 등장 |
| 2 | 75초 | 아기 방 | 없음 | 바이러스 추가, 아기 성장(기기) |
| 3 | 90초 | 거실 | 미니보스(슈퍼 세균) | 빠른 세균 추가 |
| 4 | 90초 | 거실 | 없음 | 모기 추가, 아기 성장(서기) |
| 5 | 105초 | 주방 | 보스(바이러스 대왕) | 중간보스, 분열 패턴 |
| 6 | 105초 | 주방 | 없음 | 곰팡이 추가, 아기 성장(유치원) |
| 7 | 120초 | 놀이터 | 미니보스(슈퍼 모기) | 돌진+흡혈 패턴 |
| 8 | 120초 | 놀이터 | 없음 | 비둘기 떼 추가, 아기 성장(초등) |
| 9 | 135초 | 학교 앞 | 미니보스(급식 세균왕) | 범위 공격, 독 디버프 |
| 10 | 150초 | 병원 | 최종보스(슈퍼 바이러스) | 3단계 변신, 분열+감염 패턴 |

### 난이도 곡선
- **적 스폰 속도**: 스테이지당 12% 증가
- **적 체력**: 스테이지당 15% 증가
- **적 이동속도**: 스테이지당 5% 증가
- **동시 출현 적 수 상한**: 스테이지 1(30) → 스테이지 10(150)

## 5. 플레이어 (아기)

| 스탯 | 기본값 | 성장 단계별 보너스 |
|------|--------|-----------------|
| HP | 80 | 단계당 +20 |
| 이동속도 | 1.5 | 단계당 +0.5 |
| 경험치 | 0 | 적 처치 시 획득 |
| 무적 시간 | 0.8초 | 피격 후 무적 |

### 레벨업 시스템

- 적 처치 → **별 모양 경험치** 드롭 → 수집 → 레벨업
- 레벨업 시 **3가지 선택지** 중 1개 선택:
  - 체력 회복 +20
  - 이동속도 +10%
  - 공격력 +15%
  - 공격속도 +10%
  - 투사체 크기 +15%
  - 투사체 수 +1
  - 경험치 획득량 +20%
  - 최대 HP +25

## 6. 무기 시스템 (엄마/아빠 기반)

### 무기 획득 방식
- 스테이지 진행 중 **무기 아이템**이 맵에 랜덤 스폰 (15~25초 간격)
- 아이템에 접촉하면 획득
- 이미 보유 중인 무기와 같은 종류면 **업그레이드** (최대 레벨 5)
- 새 무기면 추가 장착 (기본 **4개**, 초등학생 단계에서 **6개**)
- 슬롯이 꽉 차면 교체 여부 선택

### 무기 목록

| 무기 | 컨셉 | 기본 공격 | Lv5 효과 | 발사방식 |
|------|------|----------|----------|---------|
| **엄마 손** (시작 무기) | 엄마가 손을 뻗어 세균을 때림 | 가장 가까운 적에게 손바닥 투사체 1개 | 투사체 3개, 관통 | 자동 조준 |
| **아빠 목소리** | 아빠의 큰소리가 충격파 생성 | 주변 원형 범위 데미지 | 범위 2배, 지속 충격파 영역 | 자기 주변 |
| **아빠 슬리퍼** | 아빠 슬리퍼를 던짐 | 위로 던져 포물선 낙하 | 3개 동시, 크기 증가 | 랜덤 방향 |
| **엄마 잔소리** | 잔소리가 유도탄이 됨 | 랜덤 적에게 호밍 말풍선 | 동시 3발, 폭발 효과 | 호밍 |
| **기저귀** | 기저귀를 부메랑처럼 던짐 | 부메랑처럼 왕복 | 2개 교차 회전 | 회전 |
| **엄마 눈물** | 엄마 눈물이 번개가 됨 | 랜덤 적에게 즉시 번개 | 체인 번개 3회 | 즉발 |
| **젖병 방어막** | 젖병을 뱅글뱅글 돌려 방어 | 근접 밀어내기 + 데미지 | 넉백 강화, 범위 2배 | 자기 주변 |
| **장난감 폭탄** | 장난감이 터지며 범위 폭발 | 지면에 폭발 영역 생성 | 영역 크기 2배, 지속시간 증가 | 랜덤 위치 |

### 업그레이드 단계별 강화
- Lv1: 기본
- Lv2: 데미지 +30%
- Lv3: 공격속도 +25%
- Lv4: 투사체/범위 증가
- Lv5: 특수 효과 해금

## 7. 적 종류

| 적 | 등장 스테이지 | 행동 | 체력 | 데미지 | 외형 |
|----|-------------|------|------|--------|------|
| **일반 세균** | 1~ | 느리게 직선 추적 | 낮음 | 5 | 초록 동글이 |
| **바이러스** | 2~ | 빠르게 추적, 분열 | 낮음 | 8 | 뾰족한 빨간 공 |
| **모기** | 4~ | 빠르게 날아다님 | 매우 낮음 | 3 | 모기 |
| **곰팡이** | 6~ | 느리지만 독 데미지 | 높음 | 10 | 보라 덩어리 |

| **슈퍼 세균** | 3~ | 분열, 고체력 | 높음 | 15 | 왕 세균 |

### 보스
- **슈퍼 세균 (스테이지 3)**: 분열 공격, 일반 세균 소환

- **슈퍼 모기 (스테이지 7)**: 흡혈 돌진, 독 구름 생성
- **급식 세균왕 (스테이지 9)**: 범위 독 공격, 플레이어 감속

## 11. 아키텍처 설계 원칙

### 객체지향 + 인터페이스 기반 설계

모든 게임 요소를 **독립적인 클래스(객체)**로 분리하고, **공통 인터페이스(베이스 클래스)**를 통해 확장 가능하게 설계한다. 새로운 적, 무기, 아이템, 스테이지를 추가할 때 기존 코드를 수정하지 않고 새 클래스만 추가하면 되도록 한다.

### 베이스 클래스 인터페이스 규약

#### Entity (모든 게임 오브젝트의 부모)
```
필수 구현:
  - update(dt)        : 매 프레임 로직 업데이트
  - render(ctx)       : 캔버스에 그리기
  - getBounds()       : 충돌 판정 영역 반환
  - onDestroy()       : 제거 시 정리 작업
```

#### Enemy (적 베이스)
```
Entity를 상속하며 추가 구현:
  - hp, speed, damage  : 기본 스탯
  - movementPattern()  : 이동 패턴
  - onDeath()          : 사망 시 드롭/이펙트
  - getDropTable()     : 드롭 테이블
```

#### Boss (보스 베이스)
```
Enemy를 상속하며 추가 구현:
  - phases[]           : 페이즈별 행동 패턴
  - currentPhase       : 현재 페이즈
  - specialAttack()    : 특수 공격
  - onPhaseChange()    : 페이즈 전환 처리
```

#### Weapon (무기 베이스)
```
필수 구현:
  - name, level, maxLevel   : 무기 정보
  - cooldown, damage        : 기본 스탯
  - attack(player, enemies) : 공격 실행
  - upgrade()               : 레벨업 강화
  - getLevelDescription()   : 현재 레벨 설명
```

#### GrowthStage (성장 단계 관리)
```
필수 구현:
  - currentStage         : 현재 성장 단계 (1~6)
  - getStageStats()      : 현재 단계의 스탯 보너스
  - checkGrowth(stage)   : 클리어 스테이지로 성장 여부 확인
  - playGrowthCutscene() : 성장 컷씬 재생
  - getSpriteName()      : 현재 단계의 스프라이트 이름 반환
```

### 설계 원칙 요약

1. **단일 책임**: 각 클래스는 하나의 역할만 담당
2. **개방-폐쇄**: 기존 코드 수정 없이 새 클래스 추가로 확장
3. **인터페이스 분리**: 베이스 클래스가 명확한 계약(메서드)을 정의
4. **레지스트리 패턴**: 팩토리를 통한 동적 객체 생성
5. **데이터 주도**: 스테이지/적/무기 밸런스 값은 데이터 파일로 분리
6. **모바일 우선**: 터치 이벤트 우선, 키보드는 보조 수단
